## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wrangler CLI to Cloudflare API | Authenticated session; token scope gates what operations succeed |
| .dev.vars to filesystem | Local secrets file; must never be committed or included in build output |
| GET /api/health to D1 | Read-only constant SQL query; no user input in Phase 1 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | .dev.vars local secrets | mitigate | .gitignore must include .dev.vars entry; acceptance criteria verifies this before plan completes |
| T-02-02 | Information Disclosure | /api/health error response body | mitigate | catch block returns String(error) only — no stack trace serialization; server errors go to console.error only |
| T-02-03 | Tampering | D1 queries in health route | accept | Health route uses only a hardcoded constant SQL string with no user input and no bind parameters; SQL injection risk is zero in Phase 1 |
| T-02-04 | Elevation of Privilege | wrangler d1 create token scope | mitigate | Token scoped to Workers Scripts:Edit + D1:Edit only; broader account permissions would allow over-provisioning of cloud resources |
| T-01-SC | Tampering | npm package legitimacy | mitigate | All packages verified in RESEARCH.md Package Legitimacy Audit; no new packages installed in this plan |
</threat_model>
