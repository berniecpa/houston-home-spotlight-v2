  Rewrite .github/workflows/deploy.yml with this structure (per RESEARCH.md Pattern 7):

  name: Deploy to Cloudflare Workers

  on:
    push:
      branches:
        - main

  jobs:
    deploy:
      runs-on: ubuntu-latest
      name: Deploy
      steps:
        - uses: actions/checkout@v4

        - name: Setup Node.js
          uses: actions/setup-node@v4
          with:
            node-version: '20'
            cache: 'npm'

        - name: Install dependencies
          run: npm ci

        - name: Build (OpenNext)
          run: npx opennextjs-cloudflare build

        - name: Deploy to Cloudflare Workers
          uses: cloudflare/wrangler-action@v3
          with:
            apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
            accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
            command: deploy

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner to Cloudflare API | CLOUDFLARE_API_TOKEN passed as encrypted GitHub secret; scope must be minimum necessary |
| wrangler deploy to production Worker | CI pipeline deploys directly to live production; no staging environment in Phase 1 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | CLOUDFLARE_API_TOKEN scope in GitHub Actions | mitigate | Token scoped to Workers Scripts:Edit + D1:Edit only; full account tokens would allow arbitrary Cloudflare resource creation; Bernard verifies before CI runs |
| T-03-02 | Tampering | GitHub Actions pipeline on untrusted branches | accept | Workflow triggers only on push to main (not pull_request); no external contributor can deploy without write access to main |
| T-03-03 | Denial of Service | Failed deploy leaving inconsistent Worker state | accept | cloudflare/wrangler-action@v3 handles deploy atomicity; Cloudflare Workers deployments are versioned and can be rolled back via dashboard |
| T-01-SC | Tampering | npm package legitimacy | mitigate | No new packages installed; cloudflare/wrangler-action@v3 is the official Cloudflare-maintained GitHub Action verified in RESEARCH.md |
</threat_model>
