## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Checkout/Portal routes | Authenticated agent crosses here; uid must come from the session cookie, never the body |
| Stripe → webhook route | Untrusted external POST with no cookie; only the HMAC signature authenticates it |
| Webhook handler → D1 | Subscription state mutations driven by verified event payloads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-WH-S | Spoofing | POST /api/stripe/webhook | mitigate | constructEventAsync HMAC-SHA256 over raw body with STRIPE_WEBHOOK_SECRET; missing/invalid signature → 400 before any DB write |
| T-03-WH-T | Tampering | webhook replay | mitigate | Stripe timestamp + 300s tolerance; stripe_events event_id PK idempotency makes replays no-ops |
| T-03-CO-E | Elevation of Privilege | POST /api/stripe/checkout | mitigate | Requires valid __session cookie; uid from decodedToken only (never body); admin token → 403, no customer created |
| T-03-PO-E | Elevation of Privilege | POST /api/stripe/portal | mitigate | Session-gated; resolves customer strictly from the session uid's D1 row — no cross-agent portal access |
| T-03-SQLI | Tampering | D1 writes in handler/routes | mitigate | All queries prepare().bind() — no string concatenation (CLAUDE.md) |
| T-03-GR2 | Tampering | grace_until ms-vs-seconds | mitigate | Math.floor(Date.now()/1000)+604800; unit-tested (RESEARCH Pitfall 6) |
| T-03-IC2 | Information Disclosure | STRIPE_SECRET_KEY in routes | mitigate | Read from env at runtime; never logged; console.error logs error objects, not secrets |
</threat_model>
