<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /api/agent/profile | Untrusted profile field values; caller identity must come from the verified session, not the request body |
| Dashboard request -> layout gate | Every dashboard route gated by session + profile completeness before render |
| Worker -> D1 | Profile UPDATE uses bound parameters only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Elevation of Privilege | agent editing another agent's profile by passing a foreign uid | mitigate | PATCH route derives uid from getTokens(session), ignores any body-supplied id; WHERE id = sessionUid only |
| T-02-13 | Tampering | SQL injection via profile field values | mitigate | Parameterized D1 prepare().bind(); no string concatenation |
| T-02-14 | Elevation of Privilege | bypassing the profile gate to reach listing creation | mitigate | Gate enforced server-side in (dashboard)/layout.tsx RSC before children render; not client-only |
| T-02-15 | Information Disclosure | unauthenticated read of dashboard data | mitigate | Layout calls getTokens first and redirects to /login when absent; middleware also matches /dashboard/* |
| T-02-16 | Tampering | stored XSS via malicious photo_url rendered as <img src> | mitigate | photo_url bound as an img src attribute only (not innerHTML); React escapes; invalid URLs fall back to the placeholder circle |
</threat_model>
