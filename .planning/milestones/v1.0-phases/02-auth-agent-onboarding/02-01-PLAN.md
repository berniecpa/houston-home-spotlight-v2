<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /api/auth/session | Untrusted Firebase ID token submitted by client; must be cryptographically verified before any cookie is minted |
| Cookie -> middleware | Session cookie on every protected request; signature verified at the edge |
| Worker -> D1 | Agent row upsert uses bound parameters only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | /api/auth/session ID token | mitigate | next-firebase-auth-edge verifies the Firebase JWT via Web Crypto (Google public keys) before minting a cookie; forged tokens rejected |
| T-02-02 | Elevation of Privilege | unverified email obtaining a session | mitigate | Session route checks decodedToken.email_verified === true before Set-Cookie; returns 403 otherwise (Security Domain critical control) |
| T-02-03 | Information Disclosure | session token theft via XSS | mitigate | Cookie is HttpOnly + Secure(prod) + SameSite=Lax; JS cannot read it |
| T-02-04 | Tampering | cookie signature forgery | mitigate | cookieSignatureKeys HMAC-sign the cookie; two-key rotation supported |
| T-02-05 | Information Disclosure | Firebase private key exposure | mitigate | Private key stored as wrangler secret / .dev.vars (gitignored); newline-normalized at read; never in [vars] or git |
| T-02-06 | Tampering | SQL injection on agent upsert | mitigate | Parameterized D1 prepare().bind() — no string concatenation |
| T-02-SC | Tampering | npm installs (next-firebase-auth-edge, firebase) | accept | Both Approved in RESEARCH Package Legitimacy Audit; firebase SUS flag is documented false positive (official Google SDK, 8M+ wk downloads) — no [ASSUMED]/[SUS] requiring a blocking checkpoint |
</threat_model>
