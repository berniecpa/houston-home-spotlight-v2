## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Session cookie → billing page RSC | Page renders only for the authenticated agent; status read for the session uid only |
| Client widget → Stripe routes | CTAs POST to session-gated routes; no Stripe secret or customer id is ever exposed client-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-BW-I | Information Disclosure | BillingWidget client bundle | mitigate | Widget receives only status/grace/renewal/isAdmin — never stripe_customer_id, never the secret key |
| T-03-BP-E | Elevation of Privilege | billing/page.tsx | mitigate | Inherits the (dashboard) session+profile gate; reads status strictly for the session uid (parameterized) |
| T-03-BW-AB | Elevation of Privilege | admin notice branch | accept | Cosmetic only; authorization is enforced server-side in checkout (403 for admin) — UI hiding the CTA is defense-in-depth, not the control |
</threat_model>
