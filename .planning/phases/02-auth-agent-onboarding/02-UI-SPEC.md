## Spacing Scale

Declared values (multiples of 4 only — Tailwind default scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (p-1 / gap-1) | Icon gaps, inline label padding, input prefix padding |
| sm | 8px (p-2 / gap-2) | Compact element spacing, sidebar icon margins |
| md | 16px (p-4 / gap-4) | Default form field spacing, card internal padding |
| lg | 24px (p-6 / gap-6) | Auth card internal padding, section padding within dashboard cards |
| xl | 32px (p-8 / gap-8) | Auth card vertical padding, dashboard content area top padding |
| 2xl | 48px (p-12 / gap-12) | Auth page vertical centering top/bottom margin |
| 3xl | 64px (p-16) | Full-page vertical offset for auth card on large screens |

Exceptions:
- Touch targets: minimum 44px height/width via `.touch-target` class — applies to all sidebar nav links, form submit buttons, hamburger toggle, and resend email button.
- Sidebar width (desktop): 256px (w-64) fixed — not a spacing token but a layout constraint.
- Auth card max width: 448px (max-w-md) — constrains centered card at all breakpoints above mobile.

## Layout Contracts

Three distinct layout shells in Phase 2. Each is a separate Next.js layout file.

### 1. Auth Layout (register / login / reset-password / check-email)

- Full-viewport height: `min-h-screen`
- Background: `gray-50` (#f9fafb)
- No public Header or Footer — standalone pages
- Content: single centered `.card` (max-w-md, mx-auto)
- Card internal padding: 32px (p-8) on all sides
- Vertical centering: `flex items-center justify-center` on viewport container
- Logo: site name in `primary-900` text at top of card — not a full header bar
- Mobile: card fills width with 16px horizontal margin (mx-4); max-w-md kicks in at sm breakpoint

### 2. Dashboard Layout (/dashboard/*)

- No public Header or Footer
- Left sidebar: 256px fixed width (desktop, lg+), `primary-900` background, white text
- Sidebar nav items: 44px min height (.touch-target), 16px left padding, full-width clickable area
- Sidebar active state: `primary-700` background, full-width highlight
- Main content area: `gray-50` background, `p-8` top padding, fills remaining width
- Mobile: sidebar collapses to off-canvas drawer; hamburger toggle at top-left of content area
- Hamburger toggle: `.touch-target` (44px), `primary-900` icon on `gray-50` background
- Drawer overlay: `black/50` semi-transparent backdrop, drawer slides in from left

### 3. Admin Layout (/admin/*)

- Identical structure to dashboard layout — separate layout file, same visual pattern
- Sidebar background: `red-800` instead of `primary-900` to visually distinguish admin from agent dashboard
- Admin badge: small "ADMIN" label in `red-100` text on `red-700` background, displayed in sidebar header
- Phase 2 delivers shell only — no admin page content beyond scaffolding

## Interaction Contracts

### Form Validation (all auth forms + profile form)

- **Trigger:** `onChange` for format errors (email format, password length); `onBlur` to confirm field-level state
- **Display:** Red 14px text immediately below the input field, `text-red-600`, no icon
- **Password rule:** Minimum 8 characters — error text: "Password must be at least 8 characters"
- **Email rule:** RFC format — error text: "Enter a valid email address"
- **Required fields:** Error text: "[Field name] is required"
- **Submit button state:** Disabled (`opacity-50 cursor-not-allowed`) while form has validation errors or is submitting
- **Loading state:** Submit button shows text "Please wait..." with `opacity-75` — no spinner in Phase 2

### Server Error Banner (Firebase rejections, API errors)

- **Trigger:** On form submission failure (Firebase auth error, D1 write failure)
- **Display:** Full-width banner inside the `.card`, above the submit button
- **Styles:** `bg-red-50 border border-red-200 text-red-700 p-4 rounded-lg`
- **Dismiss:** No dismiss button — banner clears on next successful interaction or page navigation
- **Never use:** toast notifications, modals, or third-party notification libraries

### Profile Completion Progress Bar

- **Display:** Horizontal bar in WelcomeCard, `bg-gray-200` track, `bg-accent-500` fill
- **Calculation:** 5 required fields (name, photo URL, phone, brokerage, license number) — each adds 20%
- **Label:** "Profile [N]% complete" in 14px semibold, `text-gray-700`
- **CTA disabled state:** "Create your first listing" button: `opacity-50 cursor-not-allowed aria-disabled="true"` until 100%

### Sidebar Navigation

- **Active state detection:** Match `pathname` via `usePathname()` hook — active item uses `primary-700` background
- **Nav items (Phase 2):** Listings (placeholder), Leads (placeholder), Profile (active link), Billing (placeholder)
- **Placeholder items:** Render as `<span>` (not `<a>`) with `text-primary-300` and "(Coming soon)" in 12px appended
- **Mobile toggle:** `DashboardSidebar` uses `useState(false)` for drawer open state — matching `Header.tsx` pattern
- **Keyboard:** Drawer closeable via Escape key; focus returns to hamburger button on close

### Redirect Preservation

- Login page reads `?redirect=` query param — navigates to preserved path on successful auth
- Implementation: `router.push(searchParams.get('redirect') ?? '/dashboard')` after Firebase sign-in resolves

## Page-by-Page Specifications

### /register

- Title: "Create your account" (24px Merriweather semibold)
- Subtitle (14px, `text-gray-500`): "Already have an account? [Log in](/login)"
- Fields: Email (type="email"), Password (type="password"), Confirm Password (type="password")
- CTA: "Create account" — `.btn-accent` full width, `.touch-target`
- Below CTA: "By registering, you agree to our Terms of Service." (12px, `text-gray-400`, centered)
- After success: redirect to /check-email (not /dashboard — email must be verified first)

### /login

- Title: "Welcome back" (24px Merriweather semibold)
- Subtitle (14px, `text-gray-500`): "New agent? [Create an account](/register)"
- Fields: Email (type="email"), Password (type="password")
- Forgot password: "Forgot password?" link right-aligned below password field, 14px `text-primary-600`
- CTA: "Log in" — `.btn-accent` full width, `.touch-target`
- After success: redirect to `?redirect` param or `/dashboard`

### /reset-password

- Title: "Reset your password" (24px Merriweather semibold)
- Subtitle (16px, `text-gray-500`): "Enter your email and we'll send a reset link."
- Fields: Email (type="email")
- CTA: "Send reset link" — `.btn-primary` full width, `.touch-target`
- After submit: inline success message replaces form: "Check your inbox — a reset link is on its way." (no page navigation)
- Back link: "Back to login" — 14px `text-primary-600`, centered below CTA

### /check-email

- Title: "Check your inbox" (24px Merriweather semibold)
- Body: "We sent a verification link to [email]. Click the link to activate your account."
- Email address displayed in `font-medium text-primary-900`
- CTA: "Resend email" — `.btn-primary` full width, `.touch-target`
- After resend: button text changes to "Email sent!" for 3 seconds, then reverts
- Footer note: "Wrong email? [Register again](/register)" — 14px `text-gray-500`

### /dashboard (index — new agent welcome)

- Heading: "Welcome, [First Name]" (20px Merriweather semibold)
- WelcomeCard: `.card` with `p-6`, contains ProfileCompletionBar + CTA
- Empty state below card: "Your listings will appear here once you've completed your profile and created your first listing."
- CTA when profile incomplete: "Complete your profile" — link styled as `text-primary-600 underline`
- CTA when profile complete: "Create your first listing" — `.btn-accent`, disabled until 100%

### /dashboard/profile

- Heading: "Your Profile" (24px Merriweather semibold)
- Fields: Full Name (text)*, Photo URL (url)*, Phone (tel)*, Brokerage (text)*, License Number (text)*
- Asterisk (*) in `text-red-600` after label for all 5 required fields
- Photo URL preview: 64x64px `rounded-full` img next to URL input, updates `onBlur`, `bg-gray-200` placeholder circle on empty/invalid
- CTA: "Save profile" — `.btn-primary` full width, `.touch-target`
- After save success: green banner `bg-green-50 border border-green-200 text-green-700` — "Profile saved." — no navigation

### /dashboard/listings (placeholder)

### /dashboard/leads (placeholder)

### /dashboard/billing (placeholder)

## Accessibility Contract

All interactions must meet WCAG 2.1 AA.

| Requirement | Implementation |
|-------------|----------------|
| Touch targets | 44px minimum via `.touch-target` on all buttons, sidebar nav links, hamburger toggle |
| Focus ring | `focus:ring-2 focus:ring-primary-500 focus:ring-offset-2` — already in .btn-primary / .btn-accent |
| Sidebar drawer | `aria-expanded` on hamburger button; `aria-label="Navigation menu"` on drawer element |
| Form labels | `<label htmlFor="...">` matching each input `id` — no placeholder-only labeling |
| Error messages | `role="alert"` on ErrorBanner; `aria-describedby` linking input to its error paragraph |
| Disabled buttons | `aria-disabled="true"` in addition to the `disabled` HTML attribute |
| Page titles | Distinct `<title>` per auth page via Next.js `metadata` export |
| Color contrast | `primary-900` (#1e3a8a) on white: 8.6:1 ratio — exceeds AA. `red-600` on white: 5.9:1 — passes AA |
