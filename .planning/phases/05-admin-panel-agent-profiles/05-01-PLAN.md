<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| buyer/public → public reads | Untrusted public requests hit getAllListings/getListingBySlug and the lead form |
| agent client → mutation API | Authenticated agent submits create/edit/delete/pause; session cookie is the only trusted identity source |
| agent client → profile PATCH | Authenticated agent saves profile; slug derived server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | Public listing reads expose a suspended agent's listings | mitigate | AGENT_VISIBLE_SQL adds `a.is_suspended = 0` to getAllListings/getListingBySlug; single shared fragment prevents drift |
| T-05-02 | Information Disclosure | Lead form records inquiries on hidden/suspended listings | mitigate | leads listing-lookup uses AGENT_VISIBLE_SQL; suspended → "Listing not found.", no lead row |
| T-05-03 | Elevation of Privilege | Suspended agent continues mutating listings | mitigate | POST + PUT/DELETE/PATCH read agents.is_suspended for the session uid and return 403; UI banner is advisory only |
| T-05-04 | Spoofing | Slug write keyed off body-supplied uid | mitigate | Slug UPDATE uses WHERE id = session uid (getTokens), never the request body (preserves T-02-12) |
| T-05-05 | Tampering | Slug collision lets one agent overwrite another's slug | mitigate | UNIQUE constraint on agents.slug + collision-suffix loop excluding the caller's own row |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new packages introduced in this plan; no install tasks |
</threat_model>
