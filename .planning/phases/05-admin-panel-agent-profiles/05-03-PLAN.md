<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin client → admin API | Requests to /api/admin/* carry a session cookie; the admin claim is the only authorization source and must be re-verified server-side |
| non-admin → admin API | A standard agent (or attacker bypassing middleware) may call admin endpoints directly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Elevation of Privilege | Non-admin calls admin API directly (middleware bypass) | mitigate | requireAdmin re-verifies decodedToken.admin server-side in every admin route; 403 before any D1 access (defense in depth) |
| T-05-11 | Elevation of Privilege | Suspend toggle keyed off body-supplied identity | mitigate | Admin identity from getTokens (session); target agentId from the route segment; setAgentSuspended parameterized; no caller-supplied admin flag trusted |
| T-05-12 | Tampering | SQL injection via page/agentId params | mitigate | LIMIT/OFFSET bound as integers; agentId bound via prepare().bind(); no string concatenation |
| T-05-13 | Information Disclosure | Admin pages indexed by search engines | mitigate | metadata robots noindex/nofollow on /admin/agents and /admin/stats (mirrors existing /admin) |
| T-05-14 | Denial of Service | Unbounded agent list query | mitigate | Server-side pagination (page size 25) caps rows per request |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new packages introduced in this plan; no install tasks |
</threat_model>
