<objective>
Build the agent-owned listing CRUD API under /api/agent/listings: create (publishable-gated),
list-own, edit, delete, and pause/activate toggle — all with session-derived ownership and a
403 on cross-agent access. Photo URLs are persisted as ordered listing_images rows.

This is the write side of the listings system. It follows the canonical edge-API pattern from
src/app/api/agent/profile/route.ts (runtime='edge', getTokens session, parameterized D1) and
reuses the Phase 3 publishability gate as the create guard.

Purpose: Let subscribed agents manage their own listings safely.
Output: Two route files, a listings-db write-helper module, and an API test suite.
</objective>
