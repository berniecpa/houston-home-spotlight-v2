  status='processing' AND task_id IS NOT NULL AND updated_at < unixepoch() - 300 (Pitfall 6: skip
  recently-submitted jobs so the callback path is not duplicated).

  {status:'ready', videoUrl}) (same idempotent path as the callback — no double-write). On

  already on higgsfield or attempts exhausted → applyTerminalResult {status:'failed', error}. On
  { status:'processing' } → leave it (a later scan re-checks).
- Wrap each job in its own try/catch so one provider error does not abort the whole scan; console.error
  per-job failures. Side-effect-only (returns void).
- Reuse jobs.ts helpers (recordAttempt, setTaskId, applyTerminalResult) and provider.ts
  (getProviderByName); do not duplicate D1 SQL that already lives in jobs.ts where avoidable.
